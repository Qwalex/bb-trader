# Ручная настройка VK-бота (SignalsBot API)

Интеграция — Callback API сообщества ВКонтакте: входящие события приходят **POST** на NestJS-эндпоинт `POST /vk/callback`. Исходящие ответы и зеркалирование уведомлений идут через **токен сообщества** и методы VK API `5.131`.

---

## 1. Сообщество и сообщения

1. Создайте или откройте **группу** (сообщество) ВКонтакте.
2. В настройках сообщества включите **Сообщения** (раздел вроде «Управление» → «Сообщения»): чтобы пользователи могли писать сообществу, а бот — отвечать от имени сообщества.
3. При необходимости настройте приветствие и режим работы чата так, чтобы диалог с ботом был доступен целевым пользователям.

---

## 2. Callback API (вебхук)

1. В настройках сообщества откройте **Работа с API** → **Callback API** (или аналогичный раздел для Long Poll / Callback).
2. Укажите **адрес сервера** — публичный **HTTPS**-URL, по которому ваш прокси (например nginx) передаёт запросы на API.

   Путь на стороне приложения: **`/vk/callback`** (метод **POST**).

   Если открыть этот URL в браузере, уйдёт **GET** — это не то, что шлёт ВК. Раньше Nest отвечал 404 «Cannot GET»; сейчас API отдаёт короткую текстовую подсказку. Подтверждение сервера и события идут только **POST**.

   Типичный внешний URL при префиксе прокси `/trade-api`:

   `https://<ваш-домен>/trade-api/vk/callback`

   Убедитесь, что nginx (или другой reverse proxy) проксирует этот путь на тот же хост/порт, где слушает NestJS, **без обрезания тела JSON** и с достаточным таймаутом (как и для остальных тяжёлых запросов к API).

3. Включите события (типы уведомлений):

   - **`message_new`** — входящие сообщения пользователей.
   - **`message_event`** — нажатия на **callback-кнопки** (подтверждение сделок, отмена и т.д.).

4. **Подтверждение сервера**: при первой привязке ВК отправляет запрос с `type: confirmation`. API должен ответить **текстом** строкой подтверждения (см. переменную `VK_CALLBACK_CONFIRMATION` ниже). Пока строка в настройках не совпадает с тем, что ожидает ВК, адрес не подтвердится.

5. **Секретный ключ (рекомендуется)**: в панели Callback API можно задать секрет. Для обычных событий ВК присылает его в поле `secret`; в API задаётся то же значение в `VK_CALLBACK_SECRET`. Несовпадение даёт **403** (во ВК часто видно как «Invalid response code»). Запрос **`type: confirmation`** при подтверждении адреса иногда приходит **без** поля `secret` — API это учитывает; для остальных типов при заданном `VK_CALLBACK_SECRET` отсутствие или неверный `secret` по-прежнему отклоняются.

---

## 3. Ключ доступа сообщества (токен)

Нужен **токен группы** с правами, достаточными для:

- `messages` — отправка сообщений пользователям от имени сообщества, `messages.send`, ответы на `message_event` (`messages.sendMessageEventAnswer`);
- при работе с вложениями/цепочками сообщений может понадобиться доступ к сообщениям (как минимум то, что даёт ваш сценарий в админке ВК при выдаче ключа).

Создайте ключ в разделе управления сообществом (**Настройки** → **Работа с API** → ключи доступа / приложение сообщества) и сохраните его в **`VK_GROUP_ACCESS_TOKEN`**.

**Поведение без токена:** входящие callback-запросы обрабатываются, но в логах будет предупреждение, что рассылка отключена; ответы пользователю в VK и зеркалирование уведомлений **не смогут** выполняться через API.

Версия VK API в коде: **5.131**.

---

## 4. Переменные и настройки в БД

Значения читаются через общий механизм настроек: сначала **SQLite** (ключи на странице настроек / `PUT /settings`), затем **окружение** (`.env` и `ConfigService`). Для продакшена удобно хранить секреты в окружении или в БД — как принято у вас для остальных ключей.

| Ключ | Назначение |
|------|------------|
| `VK_GROUP_ACCESS_TOKEN` | Токен сообщества для исходящих сообщений и кнопок. |
| `VK_CALLBACK_CONFIRMATION` | Строка подтверждения Callback API (ответ на `type: confirmation`). |
| `VK_CALLBACK_SECRET` | Опционально, но желательно: тот же секрет, что в настройках Callback API в ВК. |
| `VK_WHITELIST` | Список **числовых id пользователей VK**, которым разрешено пользоваться ботом и **кому** дублируются уведомления из Telegram-сценария. Разделители: запятая и/или пробел. |

### Важно про whitelist

- Если **`VK_WHITELIST` пустой** — **никто** не проходит проверку для входящих команд; зеркалирование уведомлений в VK тоже **не отправляется** (список получателей пустой).
- Id пользователя ВК — это **не** никнейм и не короткая ссылка; его можно взять, например, из ответов API, виджетов для разработчиков или сервисов, если вы их используете; проще всего один раз логировать `from_id` из входящего `message_new` в защищённом окружении и добавить его в whitelist.

---

## 5. Проверка после деплоя

1. Подтвердите сервер в интерфейсе ВК (шаг с `confirmation`).
2. Напишите сообществу от пользователя, чей id есть в `VK_WHITELIST`. Должен обработаться сценарий (аналог Telegram): разбор сигналов, кнопки и т.д.
3. Нажмите callback-кнопку — должен уйти тип **`message_event`** на тот же URL.
4. На странице **`/logs`** в веб-приложении можно смотреть записи с категорией **VK** (при включённом логировании соответствующих событий).

Если ответы не приходят: проверьте токен, права ключа, что сообщения сообществу включены, что пользователь **начал диалог** с сообществом (ограничения ВК), и что peer при отправке соответствует id пользователя для личного диалога с сообществом.

---

## 6. Связка с Telegram

Зеркалирование уведомлений (подтверждения сделок, ошибки userbot и т.п.) в VK выполняется **дополнительно** к Telegram и только для id из **`VK_WHITELIST`** при наличии **`VK_GROUP_ACCESS_TOKEN`**. Логику Telegram-кода проект не заменяет.

## 7. Краткий чеклист

- [ ] Сообщения в сообществе включены.
- [ ] Callback API: URL `https://…/…/vk/callback`, POST, события `message_new` и `message_event`.
- [ ] Заданы и совпадают строка подтверждения и (если используется) секрет.
- [ ] Выдан и сохранён `VK_GROUP_ACCESS_TOKEN` с нужными правами.
- [ ] Заполнен `VK_WHITELIST` реальными id пользователей.
- [ ] Прокси пробрасывает путь к API и не режет JSON.

Детали пути в коде см. комментарий в `apps/api/src/modules/vk/vk-callback.controller.ts`.
